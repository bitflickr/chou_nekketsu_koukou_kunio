## 验收标准

- [ ] 可完整进行一场比赛，从开球到一方获胜
- [ ] 开球跳球正确发生，有胜者判定
- [ ] 内场球员淘汰后正确转移到外场
- [ ] 外场球员可通过击杀返回内场
- [ ] 一方内场全灭时正确判定胜负
- [ ] HUD 显示所有内场球员 HP 实时变化
- [ ] 比赛结束后有明确胜负反馈

## 技术要点

- **GameManager 架构**（AutoLoad）：
  ```
  States: PRE_MATCH → JUMP_BALL → IN_PROGRESS → MATCH_END
  
  职责：
  - 管理双方队伍引用
  - 跟踪内场/外场球员状态
  - 检测胜负条件
  - 控制比赛流程时序
  ```

- **规则细节**（关联 RISK-03，需在开发前确认）：
  - 发球权转换：接球成功 → 接球方；球落地 → 自由球
  - 球落地归属：落在哪方场地，该方优先拾取（AI 行为控制）
  - 外场活动范围：对方半场的上、下、对面三侧通道

- **内外场切换流程**：
  ```
  内场球员 HP → 0
  → 播放淘汰动画（闪烁）
  → 从内场列表移除
  → 添加到外场列表
  → 重新定位到外场通道
  → 更新区域约束
  ```
